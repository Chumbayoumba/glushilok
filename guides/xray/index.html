XRAY ВРУЧНУЮ

Как установить и настроить ядро Xray-core вручную через config.json и systemd — VLESS Reality без панели, для тех, кто хочет полный контроль

ЗАЧЕМ XRAY БЕЗ ПАНЕЛИ

Xray-core — это движок, на котором работает VLESS Reality (и панель 3X-UI поверх него). Ручная настройка даёт минимум лишнего на сервере (нет открытой веб-панели = меньше поверхность атаки), полный контроль над config.json и чуть меньше потребление ресурсов. Подходит тем, кто не боится терминала и JSON. Новичкам проще панель 3X-UI.

ПОШАГОВАЯ ИНСТРУКЦИЯ

1 Установи Xray-core официальным скриптом

На VPS (Ubuntu/Debian, root) поставь ядро официальным установщиком от проекта XTLS.

bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install

2 Сгенерируй ключи Reality и UUID

Reality требует пару ключей (приватный/публичный) и короткий ID. UUID — идентификатор клиента.

xray x25519        # выдаст Private key и Public key
xray uuid          # выдаст UUID клиента
openssl rand -hex 8  # короткий shortId

3 Напиши config.json

Создай /usr/local/etc/xray/config.json с inbound VLESS + Reality. Укажи приватный ключ, UUID, shortId и SNI-маскировку (dest).

{
  "inbounds": [{
    "port": 443, "protocol": "vless",
    "settings": { "clients": [{"id":"ТВОЙ_UUID","flow":"xtls-rprx-vision"}], "decryption":"none" },
    "streamSettings": {
      "network":"tcp", "security":"reality",
      "realitySettings": {
        "dest":"www.microsoft.com:443",
        "serverNames":["www.microsoft.com"],
        "privateKey":"ТВОЙ_PRIVATE_KEY",
        "shortIds":["ТВОЙ_SHORTID"]
      }
    }
  }],
  "outbounds": [{"protocol":"freedom"}]
}

4 Запусти Xray через systemd

Официальный скрипт уже создаёт сервис. Перезапусти и поставь в автозагрузку.

systemctl restart xray
systemctl enable xray
systemctl status xray   # должно быть active (running)

5 Собери ссылку vless:// для клиента

Сформируй ссылку из своих параметров: UUID, IP, порт, public key, shortId, SNI. Импортируй в Hiddify/v2rayNG.

  • Подставь свои значения вместо UUID/IP/PUBLIC_KEY/SHORTID
  • Импортируй ссылку в клиент → подключайся
  • Включи BBR для скорости
vless://UUID@IP:443?security=reality&sni=www.microsoft.com
  &pbk=PUBLIC_KEY&sid=SHORTID&flow=xtls-rprx-vision&type=tcp#MyServer
ЧАСТЫЕ ВОПРОСЫ
Xray или 3X-UI — что выбрать?
3X-UI проще (веб-панель, всё мышкой). Xray вручную — для тех, кто хочет минимум на сервере и полный контроль через config.json. Движок один и тот же.
Где смотреть ошибки?
journalctl -u xray -f покажет логи в реальном времени. Частая ошибка — опечатка в JSON или неверный privateKey.
Можно несколько клиентов?
Да, добавь несколько объектов в массив clients, каждому свой UUID. Или несколько inbounds на разных портах.
